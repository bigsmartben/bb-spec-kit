# 优惠券系统 PRD（虚拟示例）

**项目名**：优惠券中心  
**版本**：1.0  
**创建日期**：2026-03-03  
**优先级**：P1（核心功能）

---

## 项目概述

优惠券中心是电商平台的营销工具，帮助用户领取和使用优惠券，帮助运营人员创建和管理优惠券活动。

### 核心价值

- **对用户**：通过优惠券获得购物优惠，提升下单转化率
- **对运营**：灵活创建营销活动，精准控制成本
- **对平台**：提升交易额和用户活跃度

---

## 用户角色与权限

| 角色 | 权限 | 典型场景 |
|------|------|--------|
| 普通用户 | 查看、领取、使用、分享优惠券 | 买家端自助领券 |
| 运营人员 | 创建、编辑、停用、查看统计数据 | 后台平台管理 |
| 订单系统 | 验证、核销、扣减库存 | 结账时校验优惠券 |

---

## 核心功能需求

### 1. 用户领取优惠券

用户在优惠券中心浏览活跃的优惠券，点击"立即领取"按钮添加到券夹。

**约束条件**：
- 同一用户同一券最多领 1 张
- 优惠券有库存限制（全局库存、日库存）
- 优惠券有有效期限制（活动期内才可领取）
- 用户需登录且无黑名单

**成功标志**：
- 用户券夹新增 1 张优惠券
- 库存同步扣减 1

**异常场景**：
- 库存不足 → "来晚了，已抢完"
- 活动已结束 → "活动已结束，敬请期待"
- 用户已领取 → "您已领取此优惠券，请勿重复领取"
- 网络超时 → "网络异常，请重试"

---

### 2. 用户使用优惠券

用户在结账时选择已领取的优惠券应用到订单，系统验证优惠券合法性后扣减库存并计算优惠金额。

**验证规则**：
- 优惠券未过期
- 优惠券未被使用过
- 订单金额满足优惠券参加门槛
- 优惠券对应的商品/品类在订单中存在

**库存管理**：
- 使用时扣减"已使用库存"
- 防并发冲突（乐观锁/悲观锁）

**异常场景**：
- 优惠券已过期 → "优惠券已过期"
- 订单金额不满足 → "订单金额不满足使用条件，还需再买 ¥XX.XX"
- 并发冲突（库存不足） → 自动重试，超限则提示"库存不足"

---

### 3. 优惠券生命周期管理

运营人员创建优惠券活动时设置参数（券面额、库存、有效期、规则等），系统根据时间自动触发状态转换：
- **AVAILABLE**: 活动期内，用户可领取
- **CLAIMED**: 用户已领取，未使用或已过期
- **USED**: 用户已使用
- **EXPIRED**: 有效期已过
- **STOPPED**: 运营人员手动停用

## 数据模型概览

### 优惠券实体

| 字段 | 类型 | 说明 |
|------|------|------|
| coupon_id | UUID | 优惠券唯一标识 |
| name | String | 券面名称（例："满 100 减 10"） |
| amount | Decimal | 优惠金额 |
| total_stock | Integer | 总库存 |
| used_stock | Integer | 已使用数量 |
| claimed_stock | Integer | 已领取数量 |
| min_amount | Decimal | 使用门槛（订单满额） |
| start_date | DateTime | 活动开始时间 |
| end_date | DateTime | 活动结束时间 |
| status | Enum | 状态：AVAILABLE / EXPIRED / STOPPED |
| created_by | String | 创建人（运营工号） |

### 用户-优惠券关系（券夹）

| 字段 | 类型 | 说明 |
|------|------|------|
| user_id | UUID | 用户 ID |
| coupon_id | UUID | 优惠券 ID |
| claimed_at | DateTime | 领取时间 |
| used_at | DateTime | 使用时间（可为空） |
| order_id | UUID | 关联订单 ID（可为空） |
| status | Enum | 状态：AVAILABLE / USED / EXPIRED |

## Q&A（设计决策的澄清）

**Q: 同一用户在活动期内可以领取同一券多少张？**  
A: 最多 1 张。一旦领取，再次点击"领取"时提示"已领取"。

**Q: 用户领取后若不使用，优惠券什么时候失效？**  
A: 等同于其有效期。有效期到期时，状态自动从 CLAIMED 转为 EXPIRED。

**Q: 库存什么时候扣减？**  
A: 用户点击"领取"时立即扣减，不是在"使用"时扣减。（库存 = 全局库存 - 已领取与已使用）

**Q: 并发领券冲突时怎么处理？**  
A: 使用乐观锁。冲突时重试 1 次；若仍失败，则提示"库存不足，请稍后重试"。
