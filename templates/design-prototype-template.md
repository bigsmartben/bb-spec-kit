## Prototype Quality Contract *(mandatory)*

- `spec.md` remains requirements SSOT. Prototype files MUST NOT introduce normative requirements not traceable to in-scope requirement IDs or scenarios.
- Each primary `SCR-xx` in `ui/ui-spec.md` MUST map to one concrete page under `prototype/pages/`.
- Each primary page MUST include:
  - happy-path content
  - at least one non-happy state (`loading|empty|error|permission|success` as applicable)
  - screen metadata block with `screen_id`, `requirement_refs`, and `primary_actor`
- Each non-terminal primary page MUST expose at least one deterministic forward transition (`href` or deterministic JS navigation) to the next mapped screen; toast-only primary actions are prohibited.
- Non-happy states MUST be interactive and recoverable (retry/back/fix action), not text-only notes.
- Cross-screen continuity MUST be explicit via continuity keys (for example: `sessionId`, `entityId`, `mode`, `state`) passed through query params and/or localStorage, and rendered on destination pages.
- Shared mock data MUST be centralized in a single store object (for example: `window.__SPECKIT_MOCK_STORE`) and consumed by relevant pages.
- Prototype output MUST avoid unresolved placeholders (`TODO`, `TBD`, `<placeholder>`, `[insert ...]`).
